### Patterns

1. **Static Astro components** — All rendering at build time; props passed from data files.
2. **Client interactivity** — `<script is:inline define:vars={{ dataFromServer }}>` in a dedicated `.astro` file.
3. **i18n** — `data-i18n="key"` attributes + client JS updates DOM on language change.
4. **Theming** — CSS vars under `[data-theme="dark|light"]` on root. Components read vars.
5. **Typography** — Plus Jakarta Sans (body), Syne (headings), via Google Fonts.

## Summary for Planning

### Executive Summary
The codebase is minimal, clean, and ready for extension. Placeholder pages for `/models/` and `/compare` already exist. The existing benchmark + category system, CSS design tokens, and i18n infrastructure are fully reusable. Primary new additions are: 3 data files, 7 components, ECharts integration, and a scraping workflow command. No architectural changes required.

### Key Decisions
- ECharts via CDN script tag (vs npm import) — simpler for SSG
- Build-time OpenRouter API fetch — all data embedded in HTML, no runtime calls
- `benchmarks.ts` extraction — share benchmark types between home and models pages
- Extend `home.ts` translations — no separate i18n files

### Critical Files
- `src/data/home.ts` — Must preserve; add new i18n keys carefully
- `src/styles/global.css` — Design system; add new classes, don't modify existing
- `src/components/HomeBehavior.astro` — Pattern template for `ModelBehavior.astro`
- `src/layouts/BaseLayout.astro` — Add ECharts CDN here; affects all pages
